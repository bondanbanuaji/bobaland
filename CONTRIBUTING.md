## ğŸš€ How Can I Contribute?

### Reporting Bugs
- Use the [GitHub issue tracker](https://github.com/bondanbanuaji/bobaland/issues).
- Check if the issue has already been reported.
- Include details about your system (GPU, monitor setup) and relevant logs (`~/.local/share/bobaland/install.log`).

### Suggesting Enhancements
- Open an issue with the [Enhancement] tag.
- Describe the feature and why it would be useful.

### Pull Requests
1. Fork the repository.
2. Create a new branch (`git checkout -b feature/amazing-feature`).
3. Make your changes.
4. Test your changes (especially installation scripts).
5. Commit your changes (`git commit -m 'Add some amazing feature'`).
6. Push to the branch (`git push origin feature/amazing-feature`).
7. Open a Pull Request.

## ğŸ› ï¸ Development Guidelines

### Scripts
- Follow POSIX or Bash standards.
- Use the `utils.sh` library for logging and error handling.
- Keep scripts modular and idempotent.
- Add comments explaining complex logic.

### Configurations
- Keep `.config/` files organized.
- Use variables for colors and common settings.
- Ensure window rules are specific enough not to break other apps.

### Documentation
- Updates to features must be reflected in `README.md`.
- New keybindings must be added to `KEYBINDINGS.md`.
- Complex settings should be explained in `CONFIGURATION.md`.
